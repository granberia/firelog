Antigravity에 제미니 3을 사용해서 아래와 같은 프롬프트를 먹여서 웹사이트를 만들었다.

중요한 건 아래 프롬프트가 아무것도 없는 빈 작업 공간에서 위키 페이지 하나를 뚝딱 생성해냈다는 점이다.

### 프롬프트

```md
typescript와 React로 된 wikipedia 사이트를 만들고 싶어. 아래와 같은 스펙을 가진 웹사이트를 만들어 줘.
- 이 프로젝트는 백엔드를 사용하지 않을 거야. 각 문서는 빌드 시 포함된 마크다운 파일들에 포함되어 있어.
- 모든 문서들에는 프로젝트라는 대분류가 있어. 페이지 내의 좌측 사이드바에서 현재 페이지에서 표시할 프로젝트를 선택할 수 있도록 만들어 줘. 위키 페이지에 표시할 파일들은 페이지 내에서 현재 선택한 프로젝트 안에서만 검색이 가능해. 프로젝트 폴더들의 경로는 다음과 같아.
  - experiment/S3/
  - experiment/HIS/
  - 그 외에도 experiment 폴더 아래에 있는 모든 하위 폴더들이 프로젝트 폴더가 되며, 사이드바에서 표시할 때는 적절한 이름을 붙일 수 있어야 해.
- 프로젝트에 포함된 문서 파일들은 아래와 같은 경로들에 존재해. 페이지를 빌드할 때 모든 프로젝트의 문서 파일들이 포함되어야 한다는 걸 명심해.
  - experiment/HIS/KB/latest/episodes/
  - experiment/HIS/KB/latest/characters/
  - 그 외에도 latest 폴더 아래에 있는 모든 하위 폴더들의 마크다운 파일을 종합해야 해.
- 각 프로젝트 폴더마다 한글로 된 타이틀을 붙일 수 있게 config 파일을 생성해 줘. 예를 들어 S3은 "엑스페리온" 이 되어야 하고 HIS는 "패스파인더: 이스투아" 가 되어야 해.  
- 각 문서의 버전 관리는 파일이 포함된 폴더명을 통해 이루어져. 예시는 다음과 같아.
  - experiment/HIS/KB/1/characters/
  - experiment/HIS/KB/2/characters/
  - 이와 같이 latest를 제외한 과거 버전명을 가진 폴더들이 현재 문서의 과거 버전들을 포함하고 있어.
  - 문서의 history를 조회할 때 특정 버전들의 내용들끼리 git diff처럼 비교할 수 있어야 해. 가능하면 두 버전을 비교하며 하나로 취합하는 기능도 구현하자.
- 각 마크다운 파일의 시작에는 frontmatter가 포함되어 있어. 여기 있는 정보들을 마크다운으로 가공해서 표시해야 해.
- 사용자는 현재 선택된 프로젝트 내에서 문서의 제목과 내용으로 검색을 할 수 있어야 해.
  - 문서를 검색할 때 사용자가 입력한 단어와 유사한 제목을 가진 문서들이 드랍다운으로 자동완성되어야 해.
  - 검색한 단어와 문서명이 정확히 일치하면 검색 버튼을 눌렀을 때 해당 문서 페이지로 이동해야 해.
- router를 구성할 때 사용자는 특정 프로젝트와 특정 문서명으로 된 경로를 통해 문서에 바로 접근할 수 있어야 해.
- 특정 과거 버전과 비교하는 페이지에서도 어떤 버전과 비교하고 있는 지 링크에 포함해서 사용자가 바로 접근할 수 있어야 해.
- 배포 과정은 github actions로 자동화될 거야. 
- vite를 사용한 빌드 환경을 갖춰야 해. 로컬에서 테스트가 가능하도록 package.json을 구성해 줘.
- 명시된 모든 기능을 테스트할 수 있도록 프로젝트별로 위키 페이지에 넣을 예시 파일들을 몇 가지 만들어 줘. 아무 주제나 자유롭게 괜찮아. 파일들은 실제로 디렉터리 내에 존재해야 해.
```
